# Cognito audit — 2026-04-18

Comprehensive audit performed before v1.1.0 "Honest Release". Three parallel agents reviewed security, documentation accuracy, installation flow, and architectural coherence. This document is the source of truth for why each fix in v1.1.0 was made.

## Test baseline

- `pytest`: **204 passing** in 8.65s (not 201 as claimed in v1.0.0 README/CHANGELOG).
- `bats`: 22 tests (run separately, not wired into `tests/run_tests.sh`).
- Combined: **226 total**.

## Security findings

| Severity | File | Issue |
|---|---|---|
| **HIGH** | `scripts/install.sh:125` | Python heredoc without single-quoted delimiter — `$TARGET_DIR` and `$PROFILE` expand into Python source. Attacker-controlled install flags → arbitrary Python execution. |
| **HIGH** | `hooks/session-closer.sh:64,108` | `session_id` from input JSON used directly as filename without sanitization. Crafted `../../config/...` overwrites critical config. |
| **MEDIUM** | `dashboard/serve.sh:45` | `python3 -m http.server` serves entire parent directory (exposes Python source in `api/`) and may bind `0.0.0.0` depending on Python version. |
| **MEDIUM** | `dashboard/app.js` multiple | `innerHTML` used with data from `data.json` without HTML escaping. DOM-based XSS if `data.json` is poisoned (connects to the session-closer path traversal issue). |
| **MEDIUM** | `dashboard/index.html:7-8` | CDN scripts (Tailwind, Chart.js) loaded without `integrity` / `crossorigin` attributes. Supply-chain risk. |
| LOW | All hooks | Missing `-e` in `set -uo pipefail`. Intentional to keep user unblocked on hook failure, but masks Python errors from the user. |
| LOW | All hooks | Stdin read via `cat` with no size limit. Pathological input could exhaust memory. |
| LOW | All hooks | Logs append without rotation; filesystem grows unbounded over time. |
| LOW | `hooks/gate-validator.sh`, `hooks/phase-detector.sh` | User-supplied regex patterns in `_passive-triggers.json` compiled without validation/timeout. ReDoS possible (mitigated in CPython 3.11+). |
| LOW | `dashboard/api/build_data.py:99` | Exception `str(e)` embedded in output JSON — could leak filesystem paths to the dashboard. |

## Documentation accuracy

### Claims inaccurate or stale

- "201 passing tests" in README and CHANGELOG — actual 204 pytest / 226 with bats.
- "5 JSONs (estado + config)" in README directory tree — actually 6 in `config/`.
- "Python 3.8+ opcional" in INSTALL.md — actually **required** (PEP 604 syntax in `sinapsis_bridge.py` requires Python 3.10+).
- "jq recomendado, no obligatorio" in INSTALL.md — never invoked by any script; will become required in v1.1 for `settings.json` merge.
- "4 profiles" in README/ARCHITECTURE — perfiles cosméticos. `install.sh` ignores `installs.modes`/`installs.gates` from YAML (literally: `# Por simplicidad, copiamos todos los modos`).

### Missing files referenced

- `profiles/operator.hooks.json` (INSTALL.md line 30) — install instruction fails immediately.
- `scripts/update.sh` (INSTALL.md line 211) — documented but does not exist.
- `/cognition-gate off <nombre>` (INSTALL.md troubleshooting) — command does not exist in `commands/`.

### Placeholder inconsistency

- README and INSTALL use `<YOUR_GITHUB_USER>`.
- CONTRIBUTING uses `TUUSUARIO` (line 25). Inconsistent.

## Installation / integration findings

- `install.sh` never registers hooks in `~/.claude/settings.json` — manual paste from INSTALL.md required, and the referenced snippet points to a non-existent `operator.hooks.json`.
- `install.sh` is **not idempotent**: re-running overwrites `_phase-state.json`, `_operator-config.json`, commands, and mode SKILL.md files without backup.
- `uninstall.sh` does **not** touch `settings.json`. Leftover `cognito-*` hook entries point to deleted scripts → crashes on next Claude Code session until manually cleaned.
- Windows compatibility: hooks use `cygpath -m` correctly, but `install.sh` and `serve.sh` call `python3` literally. Windows Python installers default to `python`. Git Bash often has `python3` alias but not guaranteed.
- `dashboard/serve.sh` requires internet (Tailwind + Chart.js from CDN); no offline fallback.

## Architecture / code quality findings

- **Mode-injector fires on `PreToolUse`** (wrong event). Runs N times per turn instead of once, re-reading config JSONs and SKILL.md files each time. Claude sees the same mode instructions multiple times per turn → token bloat.
- **60-line truncation** (`f.readlines()[:60]`) in mode-injector drops critical sections (e.g., `Triggers de auto-activación` in `modes/divergente/SKILL.md` around line 160).
- **Metrics are lifetime counters, not per-session**. `session-closer.sh` counts lines in log files without filtering by `session_id`. Dashboard displays these as session-scoped → fundamentally incorrect.
- **`phase-detector.sh` substring match** (not word-boundary regex): "no exploremos eso" triggers "discovery" phase suggestion when user explicitly said the opposite.
- **`no-hardcode-pii` regex naive**: catches `const email = "..."` but misses JSON `"email": "..."`, object shorthand `{ email }`, Python dicts without quotes. Not trustworthy for compliance.
- **Triple source-of-truth** for mode↔phase binding: `_phases.json.defaultModes`, `_modes.json.defaultPhases`, and mode SKILL.md frontmatter. No test verifies they agree.
- **7 modes overlap**: Estratega ≈ Divergente (time-horizon is already one of Divergente's marcos). Devil's Advocate ≈ Auditor (pre-mortem vs post-mortem, same template structure). Realistic distinct count is 3-5.
- **`hookIntensity` in `_phases.json`** is declared but never read by any hook. Dead configuration.

## What v1.1.0 fixes

See `CHANGELOG.md`. In summary:

1. All 3 HIGH + 2 MEDIUM security issues patched.
2. Mode-injector moved to `UserPromptSubmit`, 60-line truncation removed.
3. Session-scoped metrics (session_id tag in every log line + rotation on close).
4. Phase-detector uses word boundaries.
5. Profile YAML honored by install.sh (modes, commands, gates filtered per profile).
6. Idempotent install (backup before overwrite, preserve `_phase-state.json`).
7. `scripts/update.sh` created.
8. `/cognition-gate` command created.
9. Auto-register and auto-unregister hooks in `settings.json` using `jq`.
10. All doc numbers synced with reality.
11. Honest "Known limitations" section in ARCHITECTURE.md (mode overlap, PII gate scope).
12. Structured JSON logs + log rotation.
13. `/cognition-status --verify` health check.
